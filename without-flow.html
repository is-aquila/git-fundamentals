Comandos Git e Respectivas Finalidades
Sem o Uso do Flow

É a parte do processo que permite registrar as alterações de um repositório git em um histórico próprio, podendo, depois, ser enviado, através do comando "push", para um repositório remoto.

git commit -m "message"

Push envia essas alterações para o repositório remoto.

git push origin branch_name

É um identificador que serve de referência que referencia um ponto específico do histórico do repositório.

git commit tag version_indicator

Branch é como uma ferramenta facilitadora para ramificação do repositório.

git branch NewBrach

Conflitos costumam ocorrer quando duas ou mais mudanças incompatíveis ocorrem no mesmo arquivo, não necessariamente ao mesmo tempo.
Caso um conflito ocorra, deve-se abrir o arquivo conflitante, retirar as marcações de conflito e fazer as alterações necessárias.
Resolvidos os conflitos, utiliza-se os seguites comandos:

git add file_name
git merge --continue
